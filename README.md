# Graphite Icon System

> Deliberate icons for enterprise software —  
> defined by rules, generated by AI.

**Graphite** is an icon system for enterprise software interfaces. Its point of difference: **design judgment is formalized as rules that AI can execute**. The designer sets the rules; the machine draws.

Built from 10+ years of shipping enterprise UI, formalized as 10 taste rules that resolve edge cases across composition, geometry, and visual weight.

## Quick start

### 1. Use the icons

Download any SVG from [`icons/`](./icons/). All use `currentColor` — recolor via CSS `color` property.

```html
<img src="icons/icn_folder.svg" style="color: #333" alt="folder">
```

### 2. Use the Claude Code skill

Copy the skill into your Claude Code skills folder:

```bash
cp -r skill ~/.claude/skills/graphite
```

Then in Claude Code, ask:

> "Generate a battery icon in Graphite style"

The skill auto-triggers, reads the rules, and produces a production-ready outlined SVG following the full Graphite methodology.

## Philosophy

Most icon systems ship a fixed library. Graphite ships something different: **the rules that produced the library**.

- The library ([`icons/`](./icons/)) shows what the system looks like
- The rules ([`docs/`](./docs/graphite-icon-system.md)) show how the system thinks
- The skill ([`skill/`](./skill/)) shows how a machine can execute the rules

This is the pivot from "designer as executor" to "designer as rule-setter."

## The 10 rules

Graphite is built on 10 taste rules that resolve edge cases:

| Rule | What it handles |
|---|---|
| §1.4 Restraint under composition | Multi-element icons (3+) simplify each element |
| §1.5 Minimal expression | Outer shape carries meaning — no redundant interior lines |
| §1.6 Structure by stroke, detail by fill | Container outlined, contents filled |
| §1.6.1 Aperture exception | Circular apertures in repeated contexts → outlined ring |
| §1.7 Proportional balance | Single element below body → bar not dot |
| §1.8 Open-bottom body forms | Vertical body shapes leave bottom open |
| §1.9 Visual weight consistency | Main elements fill 12-16 units to match system peers |
| §4.4 Stroke inside-align | Inside-aligned strokes; production form is outlined evenodd |
| §6 Radius grading | 0 / 2 / 4 / 6 / full round — match real-world softness |
| §9.2 Arrow construction | Single filled path + square perpendicular end caps |

Read the full specification at [`docs/graphite-icon-system.md`](./docs/graphite-icon-system.md).
